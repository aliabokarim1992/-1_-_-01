\section{Введение}

Целью данной лабораторной работы является изучение базовых методов анализа данных и машинного обучения на примере набора данных \textbf{Iris}. В ходе работы были выполнены первичный анализ датасета, визуализация признаков с помощью \textbf{pairplot}, построение модели \textbf{логистической регрессии} для решения задачи бинарной классификации, а также демонстрация явления \textbf{переобучения} на примере \textbf{линейной регрессии} с полиномиальными признаками.

В работе использовался датасет Iris, содержащий измерения трёх видов ириса. Для бинарной классификации были выбраны два класса: \textbf{Iris-setosa} и \textbf{Iris-versicolor}. Дополнительно были сгенерированы искусственные данные для демонстрации того, как усложнение модели влияет на её способность к обобщению.

\section{Обзор набора данных}

Набор данных \textbf{Iris} является классическим учебным датасетом, опубликованным Рональдом Фишером в 1936 году \cite{fisher}. Он содержит 150 наблюдений, относящихся к трём классам:
\begin{itemize}
    \item \textbf{Iris-setosa}
    \item \textbf{Iris-versicolor}
    \item \textbf{Iris-virginica}
\end{itemize}

Каждый объект описывается четырьмя числовыми признаками:
\begin{itemize}
    \item длина чашелистика (\textit{sepal length}), см;
    \item ширина чашелистика (\textit{sepal width}), см;
    \item длина лепестка (\textit{petal length}), см;
    \item ширина лепестка (\textit{petal width}), см.
\end{itemize}

Для решения задачи бинарной классификации в данной работе использовались только два класса --- \textbf{Iris-setosa} и \textbf{Iris-versicolor}. Класс \textbf{Iris-virginica} был исключён, чтобы привести задачу к бинарному виду.

\section{Логистическая регрессия для бинарной классификации}

\subsection{Математическая модель}

Логистическая регрессия используется для решения задач бинарной классификации и оценивает вероятность принадлежности объекта к одному из двух классов. Вероятность принадлежности к классу $y=1$ задаётся выражением:
\begin{equation}
P(y=1|X) = \sigma(W^T X + b) = \frac{1}{1 + e^{-(W^T X + b)}},
\end{equation}
где:
\begin{itemize}
    \item $X$ --- вектор признаков;
    \item $W$ --- вектор весов модели;
    \item $b$ --- смещение;
    \item $\sigma(z)$ --- сигмоидная функция.
\end{itemize}

Сигмоидная функция имеет вид:
\begin{equation}
\sigma(z) = \frac{1}{1 + e^{-z}}.
\end{equation}

После вычисления вероятности объект относится к одному из двух классов на основе выбранного порога классификации.

\subsection{Реализация модели}

В процессе выполнения работы были выполнены следующие этапы:
\begin{itemize}
    \item из исходного датасета были выбраны только классы \textbf{Iris-setosa} и \textbf{Iris-versicolor};
    \item классы были закодированы числами 0 и 1;
    \item данные были разделены на обучающую выборку (80\%) и тестовую выборку (20\%);
    \item была обучена модель логистической регрессии;
    \item качество модели было оценено на тестовой выборке с помощью метрики accuracy и матрицы ошибок.
\end{itemize}

Для тестовой выборки было получено значение точности:
\[
\text{Accuracy} = 1.0
\]

Это означает, что модель правильно классифицировала все объекты тестовой выборки.

\subsection{Матрица ошибок и анализ результата}

Для более наглядной оценки качества модели была построена матрица ошибок, представленная на рисунке \ref{fig:cm}.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.55\textwidth]{confusion_matrix.png}
    \caption{Матрица ошибок для логистической регрессии}
    \label{fig:cm}
\end{figure}

Согласно рисунку \ref{fig:cm}, модель правильно классифицировала все 10 объектов класса 0 и все 10 объектов класса 1. Ошибки классификации отсутствуют:

\subsection{Теоретические замечания}

Основная проблема переобучения заключается в том, что модель начинает запоминать обучающие данные, включая случайный шум, вместо того чтобы выявлять общие закономерности. В результате качество модели на обучающей выборке может быть высоким, а на новых данных --- существенно хуже.

Именно поэтому нельзя оценивать качество модели только на тех данных, на которых она обучалась. Для корректной оценки необходимо использовать тестовую выборку, не участвующую в процессе обучения.

\section{Демонстрация переобучения в линейной регрессии}

\subsection{Математическая модель}

Линейная регрессия описывает зависимость между входными переменными и непрерывной выходной величиной:
\begin{equation}
y = W^T X + b + \varepsilon,
\end{equation}
где $\varepsilon$ обозначает случайный шум.

Для демонстрации переобучения были сгенерированы искусственные данные с нелинейной зависимостью и шумом. После этого были построены модели различной сложности с использованием полиномиальных признаков степеней 1, 4 и 15.

\subsection{Регуляризация как способ борьбы с переобучением}

Одним из основных способов борьбы с переобучением является \textbf{регуляризация}. Суть регуляризации заключается в добавлении штрафа за слишком большие значения коэффициентов модели. Это позволяет ограничить сложность модели и сделать её более устойчивой к шуму.

Наиболее распространёнными видами регуляризации являются:
\begin{itemize}
    \item \textbf{L1-регуляризация}, способствующая занулению части коэффициентов;
    \item \textbf{L2-регуляризация}, уменьшающая значения коэффициентов и сглаживающая модель.
\end{itemize}

Использование регуляризации особенно важно при работе со сложными моделями и большим числом признаков.

\section{Заключение}

В ходе лабораторной работы был исследован набор данных \textbf{Iris}, выполнен его первичный и визуальный анализ, а также решена задача бинарной классификации с помощью \textbf{логистической регрессии}. Полученные результаты показали, что выбранные классы \textbf{Iris-setosa} и \textbf{Iris-versicolor} хорошо разделяются в пространстве признаков, а модель обеспечивает \textbf{100\% точность} на тестовой выборке.

Кроме того, на искусственно сгенерированных данных было показано, как изменение сложности модели влияет на качество аппроксимации и способность к обобщению. Полином первой степени продемонстрировал недообучение, полином четвёртой степени --- наиболее адекватное приближение, а полином пятнадцатой степени --- явное переобучение.

Основные выводы по работе:
\begin{itemize}
    \item визуализация данных позволяет оценить разделимость классов ещё до обучения модели;
    \item логистическая регрессия является эффективным методом бинарной классификации;
    \item качество модели необходимо оценивать на тестовой выборке;
    \item чрезмерное усложнение модели приводит к переобучению;
    \item регуляризация является одним из основных способов уменьшения переобучения.
\end{itemize}
